# 端口转发 - 内网网卡

## 什么是内网网卡端口转发？

VPS 通常有两个网卡和两个 IP：
- **内网网卡**（如 ens3）：内网 IP（如 10.0.0.5）
- **公网网卡**（如 eth0）：公网 IP（如 1.2.3.4）

有些 VPS 只能通过内网 IP 互相访问，这时需要指定内网网卡进行转发。

**使用场景**：
- 你的 VPS 内网 IP 是 10.0.0.5（网卡 ens3）
- 你想将内网 IP 的 80 端口转发到另一台 VPS 的内网 IP 10.0.0.10:8080

## 查看网卡和 IP

```bash
# 查看所有网卡和 IP
ip addr

# 输出示例：
# 1: lo: <LOOPBACK,UP,LOWER_UP>
#     inet 127.0.0.1/8
# 2: ens3: <BROADCAST,MULTICAST,UP,LOWER_UP>
#     inet 10.0.0.5/24        # 内网 IP
# 3: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP>
#     inet 1.2.3.4/32         # 公网 IP
```

## 前置准备

```bash
# 开启 IP 转发
echo 1 > /proc/sys/net/ipv4/ip_forward
echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf
sysctl -p
```

## 完整示例

### 场景：通过内网网卡 ens3 转发 80 端口到 10.0.0.10:8080

```bash
# 1. 创建 nat 表
nft add table ip nat

# 2. 创建 prerouting 链
nft add chain ip nat prerouting { type nat hook prerouting priority -100 \; }

# 3. 创建 postrouting 链
nft add chain ip nat postrouting { type nat hook postrouting priority 100 \; }

# 4. 指定内网网卡进行转发
nft add rule ip nat prerouting iifname "ens3" tcp dport 80 dnat to 10.0.0.10:8080

# 5. 源地址转换
nft add rule ip nat postrouting oifname "ens3" masquerade
```

## 命令解释

| 命令部分 | 含义 |
|---------|------|
| `iifname "ens3"` | 只处理从 ens3 网卡进来的流量 |
| `oifname "ens3"` | 只处理从 ens3 网卡出去的流量 |
| `dnat to 10.0.0.10:8080` | 转发到内网 IP |
| `masquerade` | 自动源地址转换 |

## 更多示例

### 转发到另一台机器（不限网卡）

```bash
# 转发到内网另一台机器
nft add rule ip nat prerouting tcp dport 80 dnat to 192.168.1.100:8080
nft add rule ip nat postrouting ip daddr 192.168.1.100 masquerade
```

### 同时监听内网和公网

```bash
# 内网网卡转发
nft add rule ip nat prerouting iifname "ens3" tcp dport 80 dnat to 10.0.0.10:8080

# 公网网卡转发
nft add rule ip nat prerouting iifname "eth0" tcp dport 80 dnat to 10.0.0.10:8080

# 统一的 masquerade
nft add rule ip nat postrouting masquerade
```

### 转发 UDP 端口

```bash
nft add rule ip nat prerouting iifname "ens3" udp dport 53 dnat to 10.0.0.10:53
```


## 验证转发

```bash
# 查看规则
nft list table ip nat

# 从另一台内网机器测试
curl http://10.0.0.5:80
# 应该能访问到 10.0.0.10:8080 的服务
```

## 常见问题

**Q: 如何确定使用哪个网卡？**
- 运行 `ip addr` 查看所有网卡
- 内网 IP 通常是 10.x.x.x 或 172.16.x.x 或 192.168.x.x
- 网卡名称常见：ens3, eth0, eth1

**Q: 转发不生效？**
- 检查 IP 转发：`cat /proc/sys/net/ipv4/ip_forward`（应该是 1）
- 检查网卡名称是否正确：`ip addr`
- 检查目标机器是否可达：`ping 10.0.0.10`

**Q: 要不要指定网卡？**
- 如果只有一个网卡，可以不指定
- 如果有多个网卡（内网+公网），建议指定避免混乱
