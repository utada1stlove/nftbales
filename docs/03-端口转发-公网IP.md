# 端口转发 - 公网网卡

## 什么是公网网卡端口转发？

通过公网网卡进行端口转发，可以转发到公网 IP 或内网 IP。

**使用场景**：
- 你的 VPS 有公网 IP 1.2.3.4（网卡 eth0）
- 你想将公网 IP 的 80 端口转发到另一台服务器

## 查看网卡和 IP

```bash
# 查看所有网卡和 IP
ip addr

# 输出示例：
# 2: ens3: <BROADCAST,MULTICAST,UP,LOWER_UP>
#     inet 10.0.0.5/24        # 内网 IP
# 3: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP>
#     inet 1.2.3.4/32         # 公网 IP
```

## 前置准备

```bash
# 开启 IP 转发
echo 1 > /proc/sys/net/ipv4/ip_forward
echo "net.ipv4.ip_forward=1" >> /etc/sysctl.conf
sysctl -p
```

## 完整示例

### 场景：通过公网网卡 eth0 转发 80 端口到 5.6.7.8:8080

```bash
# 1. 创建 nat 表
nft add table ip nat

# 2. 创建 prerouting 链
nft add chain ip nat prerouting { type nat hook prerouting priority -100 \; }

# 3. 创建 postrouting 链
nft add chain ip nat postrouting { type nat hook postrouting priority 100 \; }

# 4. 指定公网网卡进行转发
nft add rule ip nat prerouting iifname "eth0" tcp dport 80 dnat to 5.6.7.8:8080

# 5. 源地址转换
nft add rule ip nat postrouting oifname "eth0" masquerade
```

## 命令解释

| 命令部分 | 含义 |
|---------|------|
| `iifname "eth0"` | 只处理从 eth0 网卡进来的流量 |
| `oifname "eth0"` | 只处理从 eth0 网卡出去的流量 |
| `dnat to 5.6.7.8:8080` | 转发到目标 IP |

## 更多示例

### 不指定网卡（转发所有流量）

```bash
# 不限制网卡，所有流量都转发
nft add rule ip nat prerouting tcp dport 80 dnat to 5.6.7.8:8080
nft add rule ip nat postrouting masquerade
```

### 转发到内网 IP（通过公网网卡）

```bash
# 公网访问转发到内网机器
nft add rule ip nat prerouting iifname "eth0" tcp dport 80 dnat to 10.0.0.10:8080
nft add rule ip nat postrouting masquerade
```


### 只转发特定来源IP

```bash
# 只转发来自 100.100.100.100 的流量
nft add rule ip nat prerouting ip saddr 100.100.100.100 tcp dport 80 dnat to 5.6.7.8:8080
```

### 转发多个端口

```bash
# 转发 80 和 443
nft add rule ip nat prerouting iifname "eth0" tcp dport 80 dnat to 5.6.7.8:80
nft add rule ip nat prerouting iifname "eth0" tcp dport 443 dnat to 5.6.7.8:443
```

## 验证转发

```bash
# 查看规则
nft list table ip nat

# 从外部测试
curl http://你的公网IP:80
# 应该能访问到目标服务器的内容
```

## 常见问题

**Q: 内网网卡和公网网卡有什么区别？**
- 内网网卡：IP 通常是 10.x.x.x、172.16.x.x、192.168.x.x
- 公网网卡：其他 IP
- 用 `ip addr` 查看

**Q: 要不要指定网卡？**
- 单网卡：不需要指定
- 多网卡：建议指定，避免流量走错网卡

**Q: 转发不生效？**
- 检查 IP 转发：`cat /proc/sys/net/ipv4/ip_forward`
- 检查网卡名称：`ip addr`
- 检查目标是否可达：`ping 目标IP`
