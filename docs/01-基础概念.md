# nftables 基础概念

## 什么是 nftables？

nftables 是 Linux 内核的防火墙工具，用于控制网络流量。它是 iptables 的替代品，更简洁、更高效。

## 为什么用 nftables 而不是 iptables？

- **语法更简单**：规则更容易理解和编写
- **性能更好**：处理规则更快
- **统一管理**：IPv4、IPv6、ARP 等都用同一套命令

## 核心概念

nftables 有三层结构：

```
table（表）
  └── chain（链）
        └── rule（规则）
```

### 1. Table（表）

表是最外层容器，用于组织规则。常用的表类型：

- **ip**：处理 IPv4 流量
- **ip6**：处理 IPv6 流量
- **inet**：同时处理 IPv4 和 IPv6（推荐）
- **nat**：网络地址转换（端口转发用这个）

### 2. Chain（链）

链是规则的集合，决定何时触发规则。常用的链类型：

- **input**：进入本机的流量
- **output**：从本机发出的流量
- **forward**：经过本机转发的流量
- **prerouting**：流量进入路由前处理（端口转发用这个）
- **postrouting**：流量离开路由后处理（端口转发用这个）

### 3. Rule（规则）

规则定义具体的操作，例如：
- 允许某个端口
- 禁止某个 IP
- 转发端口

## 基本命令格式

```bash
# 查看所有规则
nft list ruleset

# 添加表
nft add table inet mytable

# 添加链
nft add chain inet mytable mychain { type filter hook input priority 0 \; }

# 添加规则
nft add rule inet mytable mychain tcp dport 22 accept
```

## 规则持久化

nftables 的规则重启后会丢失，需要保存：

```bash
# 保存当前规则
nft list ruleset > /etc/nftables.conf

# 启用开机自动加载
systemctl enable nftables
```
